Detailed Description

Custom hashing implementation for use in std::unordered_map.

Based on this article.

The default std::unordered_map implementation uses uint64_t hashes, which makes collisions possible by having many Accounts and distributing them in a way that they have the same hash across all nodes.

This struct is a workaround for that, it's not perfect because it still uses uint64_t, but it's better than nothing since nodes keep different hashes.
