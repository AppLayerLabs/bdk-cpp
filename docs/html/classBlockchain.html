Detailed Description

Master class that represents the blockchain as a whole.

Contains and acts as the middleman of every other part of the core and net protocols. Those parts interact with one another by communicating through this class.

◆ start()

void Blockchain::start ( )

Start the blockchain.

Initializes P2P, HTTP and Syncer, in this order.

When starting the binary, the first thing to setup is the gRPC server, as the AvalancheGo daemon node will be waiting for the gRPC server to answer on the terminal.

Register "service" as the instance through which we'll communicate with clients. In this case it corresponds to a synchronous service.

Assemble the server and send the address of the gRPC server to the AvalancheGo Daemon.

Wait for the server to shutdown. Note that some other thread must be responsible for shutting down the server for this call to ever return.

◆ stop()

void Blockchain::stop ( )

Stop/shutdown the blockchain.

Stops Syncer, HTTP and P2P, in this order (reverse order of start()).
